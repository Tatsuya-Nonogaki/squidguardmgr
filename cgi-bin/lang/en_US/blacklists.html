This is a list of blocklist file groups found in the SquidGuard DBHOME directory. A group actually is a directory containing an each or none of domains, URLs and expressions files.

Note that if you have created subdirectories under those directories, to subclassify your blocklists for example, SquidGuard Manager will not work and may corrupt your squidguard.conf file!
Note also that if you have defined symbolic links into that directory, SquidGuard Manager will not work, and they will not be displayed here properly.

To start editing a group of files, click on the Edit icon of your choice. There you can add/remove entries to/from them. You can also give an alias name and descriptive notes to the group.

The Rebuild all databases button allows you to reconstruct all Berkeley database of all the lists. It's just like running a "squidguard -C all".

To add a new list, click on the Create a list button and give it a name. The name must only consist of alphanumeric, dash '-' and underscore '_' characters.

To delete a whole list directory, click on the rightmost Trash icon of the row. Then you will be prompted before actual deletion takes place.

Sometimes it is difficult to know where a blacklisted domain or URL is stored. Use the Search form to know in witch list it is.
